Why Messages Are Failing

Borrowed from the original PIQI Engine experience: give people a simple explanation of what the failed checks are telling them.

Performed Date Missing PIQI expects this field for lab timing context. Missing values reduce trust in the result and drive failures.
Result Status Missing This is a required operational field. When absent, the dashboard should explain that the result may be clinically incomplete.
Skipped Checks Are Not Always Bad Some checks are skipped because the incoming value type does not apply. A new user should not confuse “skipped” with “failed.”
Passed = requirement met
Failed = action needed
Skipped = not applicable or missing context
Drill in = view per-attribute detail

Recommended Next Actions

Instead of making a new user interpret everything alone, the dashboard can suggest where to go next.

1. Review the worst-performing source first Open the source details page for “POC Device Feed” and confirm whether the same missing fields recur across all messages.
2. Drill into failed checks Filter to failed checks only and focus on required fields before terminology or optional attributes.
3. Compare batch vs source trend Use the trend chart to decide whether this is a one-off bad batch or a persistent feed-quality problem.
